Introduction

Git is a powerful, distributed version control system that enables efficient management of source code for projects. Think of Git as a library where all versions of your work are stored. Instead of a single copy of a book (a version of your code), you have an entire shelf with different editions, each with its own date and description of changes. This allows you to revert to any point in the project's history, compare different versions, and even work on multiple ideas simultaneously without risking data loss.

Git as a Version Control System

A version control system (VCS) is a tool used to track changes in files, most commonly in software development projects. There are three main types of version control:

  • Local: All changes are stored on a local computer. This is the simplest but least secure method.
  • Centralized: There is a single central server where all file versions are stored. Each user has access to the same copy.
  • Distributed: Each user has a complete copy of the repository on their computer. This allows for offline work and easier collaboration within teams. Git is an example of such a system.
How to install

  • Linux:

    sudo apt install git # Debian

    or

    dnf install git # Fedora
  • Windows: Get the installer from the official Git website and follow the steps.

  • macOS: You can install Git using Homebrew:

    brew install git

    Or you can download the installer from the official website.

New repository

A repository is where all the information about a project is stored, including files, change history, and configuration. To create a new, local repository, navigate to the desired directory in your terminal and execute the following command:

git init

or

git init <repository name>
First commit

A commit is a snapshot of your project at a specific point in time. To stage all new or modified files for the initial commit, use:

git add *

Then we commit these changes to the repository.

git commit -m "First commit"

Where -m is used to add a brief description of the changes.

Creating a branch

A branch is a separate version of your repository. It's like creating a new working copy of your project. To create a new branch, run:

git branch <branch name>

To switch to another branch:

git checkout <branch name>
