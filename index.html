Introduction

Git is a powerful, distributed version control system designed to efficiently manage source code for various projects. Imagine Git as a library where all versions of your work are stored. Instead of having a single copy of a book (version of code), you have an entire shelf with different editions, each with its own date and change description. This allows you to revert to any point in your project's history, compare different versions, and even work on multiple ideas simultaneously without risking data loss.

VCS

A version control system (VCS) is a tool used to track changes in files, most commonly in software development projects. There are three main types of version control:

  • Local: All changes are stored on the local computer. This is the simplest but least secure method.
  • Centralized: There is a single central server where all file versions are stored. Each user has access to the same copy.
  • Distributed: Each user has a full copy of the repository on their computer. This allows for offline work and easier collaboration within teams. Git is such a system.
Installation

  • Linux:

    sudo apt install git # Debian

    or

    dnf install git # Fedora
  • Windows: Download the installer from the official Git website and follow the instructions.

  • macOS: You can install Git using Homebrew:

    brew install git

    Or download the installer from the official website.

Repository

A repository is where all project information is stored, including files, change history, and configuration. To create a new, local repository, navigate to the desired directory in your terminal and execute the command:

git init

or

git init <repository name>

This is like creating a new, empty shelf in our library.

Commit

A commit is a snapshot of the changes in the repository. It's like adding a new book to the shelf. To add all new or modified files to the staging area, run:

git add *

Then save these changes to the repository:

git commit -m "First commit"

Where -m is used to add a brief description of the changes.

Branch

A branch is an independent line of development for a project. Think of it as a new shelf in the library where you're working on a new version of a book. To create a new branch, execute:

git branch <branch name>

To switch to a different branch:

git checkout <branch name>
Merge

A merge is the combination of two branches. It's like merging two books into one. To merge the new-branch into the main branch (main), first switch to the main branch:

git checkout main

Then perform the merge:

git merge new-branch

If there are conflicts, you'll need to resolve them manually.
